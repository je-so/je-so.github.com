Waitlist

Allows to let threads wait for a certain condition.  If the condition is signaled the first thread in the waiting list is woken up if and the thread argument is set to the value given in signaling function.

struct waitlist_t

Implements facility where threads can wait for a certain condition.  Similar to semaphore_t.  The difference is that a thread’s Â»commandÂ« parameter is set to a specific value.  Therefore a woken up thread knows what to do next.

This object is thread safe.

Wakes up all waiting threads and frees all resources.  Make sure that no more other thread is trying to call wait_waitlist if free_waitlist is processing or was already called.

query

isempty_waitlist

extern bool isempty_waitlist(waitlist_t *wlist)

Returns true if no thread is waiting else false.  Before testing the lock on the list is acquired.

But in case more than one thread calls trywakeup_waitlist you can not be sure that trywakeup_waitlist does not return EAGAIN even if isempty_waitlist returns false.

nrwaiting_waitlist

extern size_t nrwaiting_waitlist(waitlist_t *wlist)

Returns the number of threads waiting on this list.  Before testing the lock on the list is acquired.

But in case more than one thread calls trywakeup_waitlist you can not be sure that trywakeup_waitlist does not return EAGAIN even if nrwaiting_waitlist returns a value greater 0.

change

wait_waitlist

extern int wait_waitlist(waitlist_t *wlist)

Suspends the calling thread until some other calls trywakeup_waitlist.  The waiting threads are woken up in FIFO order.  The calling thread enters itself as last in the waiting list and then it suspends itself.  See also <suspend_thread>.

trywakeup_waitlist

extern int trywakeup_waitlist(waitlist_t *wlist,
task_callback_f task_main,
callback_param_t *start_arg)

Tries to wake up the first waiting thread.  If the list is empty EAGAIN is returned and no error is logged.  If the list is not empty the argument thread_t.task of the first waiting thread is set to task_main and start_arg.  Th first thread is removed from the list.  It is then resumed.  See also <resume_thread>.
